# AI 智能辅助

PATH 插件集成 AI 服务，可根据测试步骤注释智能匹配方法并生成代码。

## 功能概述

- 自动解析测试文件中的步骤和预期注释
- 扫描项目中所有可用的测试方法
- 使用 AI 理解步骤语义，匹配最合适的方法
- 自动生成方法调用代码
- 生成可视化执行报告

## 配置说明

在 VSCode 设置中（`Ctrl+,`，搜索 `path.ai`）配置：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| path.ai.url | AI 服务地址 | `http://127.0.0.1:8045/v1` |
| path.ai.apiKey | API 密钥 | - |
| path.ai.model | 模型名称 | `gemini-3-flash` |

## 使用步骤

### 1. 准备测试文件

确保测试文件中包含步骤注释，格式如下：

```python
import pytest

@pytest.fixture(autouse=True, scope='function')
def setup_teardown(self, logic: LogicExport):
    # 前置步骤 1: 打开项目
    # 前置步骤 2: 准备测试数据
    yield

def test_login_001(self, logic: LogicExport):
    """用户登录功能测试"""
    # 步骤 1: 打开登录页面
    # 预期 1: 页面正常显示
    
    # 步骤 2: 输入用户名 admin
    # 预期 2: 输入成功
    
    # 步骤 3: 输入密码 123456
    # 预期 3: 输入成功
    
    # 步骤 4: 点击登录按钮
    # 预期 4: 登录成功
```

### 2. 触发 AI 生成

1. 打开测试文件（`test_*.py`）
2. 在编辑器中右键
3. 选择 **AI Generation**

### 3. 查看结果

- 生成的代码自动插入到对应步骤下方
- 前置步骤代码会插入到 `setup_teardown` fixture 中
- 辅助视图中显示执行报告

## 步骤注释格式

### 标准格式

```python
# 前置步骤 N: 前置操作描述
# 步骤 N: 步骤描述
# 预期 N: 预期结果
```

其中 `N` 为步骤序号。

### 示例

```python
# 前置步骤 1: 启动工程
# 步骤 1: 点击【添加】按钮
# 预期 1: 弹出添加对话框

# 步骤 2: 在【名称】输入框中输入 test
# 预期 2: 输入成功
```

**提示**：使用 `【】` 包裹关键词可以提高匹配准确性。

## 匹配机制

### 候选方法收集

插件会扫描 `method` 目录下的所有方法，包括：

- 普通方法（用于步骤操作）
- 断言方法（用于预期验证）

### 搜索权重

以下内容权重更高：

- `codePath`（方法路径）
- `【】` 包裹的内容
- 方法文档字符串

### AI 匹配

将步骤描述和候选方法列表发送给 AI，AI 返回：

- 匹配的方法路径
- 方法参数

## 生成代码示例

**输入**：

```python
# 步骤 1: 点击【添加】按钮
# 预期 1: 弹出添加对话框
```

**输出**：

```python
# 步骤 1: 点击【添加】按钮
# 预期 1: 弹出添加对话框
logic.editor.dialog.add_btn.click()
logic.asserts.dialog_visible()
```

## 执行报告

AI 生成完成后，辅助视图会显示 HTML 格式的报告：

| 状态 | 说明 |
|------|------|
| Success | 成功匹配方法并生成代码 |
| Failed | 未找到合适的方法或 AI 调用失败 |
| Skipped | 该步骤已有代码，跳过生成 |

报告中会显示每个步骤的处理结果和详细信息。

## 常见问题

### AI 服务连接失败

- 检查 AI 服务地址是否正确
- 确认 API 密钥有效
- 验证网络连接

### 生成的代码不准确

- 优化步骤描述，使用更精确的术语
- 在关键词外添加 `【】`
- 确保方法树中有对应的方法
- 手动调整生成的代码

### 没有生成代码

- 检查步骤注释格式是否正确
- 确认方法树中有可匹配的方法
- 查看辅助视图中的错误信息

## 注意事项

- AI 生成的代码可能不完全准确，建议人工检查
- 已有代码的步骤会自动跳过，避免重复生成
- 可以多次运行 AI 生成，只处理未实现的步骤
