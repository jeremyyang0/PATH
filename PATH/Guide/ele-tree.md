# 测试元素树

测试元素树用于解析和展示项目中所有的 Ele 类变量，支持搜索、拖拽、跳转等操作。

## 功能概述

- 自动解析项目中所有 `*_ele.py` 文件
- 按照 Ele 变量的层级关系（hierarchy）组织树形结构
- 支持中文描述和英文变量名双语显示
- 提供搜索、刷新、展开/收起等操作

## 解析机制

### 支持的文件格式

插件会扫描项目中所有以 `_ele.py` 结尾的文件，解析其中的 Ele 类变量。

### 解析内容

| 属性 | 说明 |
|------|------|
| name | 变量名（英文） |
| desc | 描述（中文） |
| hierarchy | 层级关系，用于组织树形结构 |
| line | 定义所在行号 |
| arguments | Ele 构造参数 |

### 示例

```python
class LoginPage(Ele):
    username_input = Ele(By.ID, 'username', desc='用户名输入框')
    submit_btn = Ele(By.XPATH, '//button[@type="submit"]', desc='提交按钮')
```

解析后的树形结构：

```
page
└── login
    ├── 用户名输入框 [username_input]
    └── 提交按钮 [submit_btn]
```

## 操作说明

### 浏览元素

- **展开/收起**：点击文件夹图标展开或收起子节点
- **查看详情**：鼠标悬停在元素上，显示完整信息（变量名、描述、文件路径、行号）

### 搜索元素

1. 点击树形视图顶部的搜索按钮
2. 输入关键词
3. 实时显示匹配结果
4. 搜索时自动展开匹配的节点
5. 点击清除按钮恢复完整视图

### 跳转到源码

- **双击**元素节点，自动打开对应的 `*_ele.py` 文件
- 光标跳转到变量定义行
- 目标行显示在编辑器中心

### 拖拽元素

1. 在元素树中找到目标元素
2. 按住鼠标左键拖拽到编辑器
3. 松开鼠标，自动插入变量路径

**生成规则**：根据树形结构生成完整路径，如 `page.login.submit_btn`

### 右键菜单

在元素节点上右键，可选择：

| 操作 | 生成代码 |
|------|----------|
| 点击 | `.click()` |
| 双击 | `.double_click()` |

代码会自动插入到当前 `atomic.py` 文件的方法末尾。

### 批量操作

1. 按住 `Ctrl` 选择多个元素
2. 右键选择操作类型
3. 所有选中的元素依次生成代码

## 状态保持

插件会自动保存以下状态：

- 展开/收起的节点
- 搜索关键词

切换文件或刷新后，状态会自动恢复。

## 刷新视图

当 `*_ele.py` 文件发生变化时：

1. 点击顶部刷新按钮
2. 插件重新扫描和解析文件
3. 自动保持展开状态

## 常见问题

### 元素没有显示

- 检查文件名是否以 `_ele.py` 结尾
- 确认 Ele 变量定义格式正确
- 点击刷新按钮重新扫描
