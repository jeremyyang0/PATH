## 快速入门

### 第一步：安装插件

1. 下载最新的 `.vsix` 文件
2. 打开 VSCode，按 `Ctrl+Shift+P` 打开命令面板
3. 输入 "Install from VSIX"，选择下载的文件

### 第二步：配置插件

打开 VSCode 设置（`Ctrl+,`），搜索 `path`，配置以下信息：

#### 禅道配置（可选）

- **Host**：禅道服务器地址（如 `10.0.18.30`）
- **Username**：禅道用户名
- **Password**：禅道密码

#### 应用配置（可选）

- **App Name**：默认应用名称（如 `Logic`），配置后创建用例时自动填充

#### AI 配置（可选）

- **AI URL**：AI 服务地址
- **API Key**：AI API 密钥
- **Model**：模型名称

### 第三步：打开树形视图

点击 VSCode 左侧活动栏的 **PATH** 图标，可以看到：

- **测试元素树**：显示所有 Ele 变量
- **测试方法树**：显示所有测试方法

## 功能详解

### 1. 测试元素树操作

#### 浏览元素

- **展开/收起**：点击文件夹图标展开或收起子节点
- **查看详情**：鼠标悬停在元素上，查看完整信息（变量名、描述、文件路径）

#### 搜索元素

1. 点击树形视图顶部的 **搜索** 按钮
2. 输入关键词（支持中文描述或英文变量名）
3. 实时显示匹配结果，自动展开匹配的节点
4. 点击 **清除搜索** 按钮恢复完整视图

#### 跳转到源码

- **单击**元素节点：自动打开对应的 `*_ele.py` 文件，并跳转到变量定义行
- 目标行会显示在编辑器中心，光标定位到行首

#### 拖拽元素

1. 在元素树中找到目标元素
2. **拖拽**到编辑器中
3. 自动生成变量引用路径（如 `page.dialog.submit_btn`）

#### 右键菜单操作

1. **右键点击**元素节点
2. 选择操作类型：
   - **点击**：生成 `.click()` 代码
   - **双击**：生成 `.double_click()` 代码

#### 批量添加操作

1. 选择任意非叶子节点
2. **右键**点击选择添加的操作类型（点击、双击、全部）
3. 所有该节点下的所有叶子节点都会生成对应的操作方法

### 2. 测试方法树操作

#### 浏览方法

- **包结构**：按照 Python 包结构展示 method 目录
- **中文包名**：英文包名自动替换为中文（从 `__init__.py` 中读取）
- **方法列表**：每个模块下显示所有方法，包含方法文档

#### 搜索方法

1. 点击树形视图顶部的 **搜索** 按钮
2. 输入关键词（方法名、文档字符串、文件路径）
3. 实时显示匹配结果
4. 点击 **清除搜索** 恢复完整视图

#### 双击跳转

- **双击文件夹节点**：打开该包的 `__init__.py` 文件
- **双击方法节点**：跳转到方法定义位置

#### 拖拽方法

1. 找到目标方法
2. **拖拽**到编辑器中
3. 自动生成方法调用路径

### 3. 创建测试用例

#### 操作步骤

1. 在 VSCode **资源管理器**中，右键点击目标文件夹或文件
2. 选择 **Create Case (创建用例)**
3. 输入禅道用例 ID（可选）：
   - **输入 ID**（如 `12345`）：自动从禅道拉取用例信息
   - **回车跳过**：生成基础空模板
4. 输入应用名称（如已配置 `path.appName`，自动跳过）
5. 插件自动生成测试文件并打开

#### 自动生成内容

- **文件名**：`test_<folder_name>_<id>.py`（ID 自动递增）
- **导入语句**：自动生成符合项目规范的 import
- **类结构**：生成标准的测试类和方法
- **步骤注释**：将禅道步骤转换为代码注释

#### 示例

假设在 `login` 文件夹中创建用例：

**输入**：
- 禅道 ID：`12345`
- 应用名称：`Logic`

**生成文件**：`test_login_001.py`

```python
# -*- coding: utf-8 -*-
"""
测试用例文件: test_login_001.py
禅道ID: 12345
用例标题: 打开文件功能测试
"""
from method.logic import LogicExport
from case.base_case import BaseCase

class TestLogic(BaseCase):

    def test_login_001(self, logic: LogicExport):
        """打开文件功能测试"""
        # 步骤 1: 打开logic
        # 预期 1: logic正常打开
        
        # 步骤 2: 打开文件xxx.sch
        # 预期 2: 文件xxx.sch打开成功
```

### 4. AI 智能生成代码

#### 前提条件

- 已创建测试文件，包含步骤注释
- 已配置 AI 服务（URL、API Key、Model）

#### 操作步骤

1. 打开包含步骤注释的测试文件（`test_*.py`）
2. **右键**编辑器，选择 **AI Generation**
3. AI 自动解析步骤，匹配合适的方法
4. 生成的代码自动插入到对应步骤下方
5. 查看辅助视图中的执行报告

#### 步骤注释格式

```python
# 步骤 1: 打开登录页面
# 预期 1: 页面正常显示
```

AI 会识别 "步骤" 和 "预期" 关键词，提取测试步骤描述。

#### 执行报告

报告显示每个步骤的处理结果：

- **Success**（成功）：找到方法并生成代码
- **Failed**（失败）：未找到合适的方法或 AI 调用失败
- **Skipped**（跳过）：该步骤已有代码，跳过生成

### 5. 调试测试用例

#### 自动配置调试

插件启动时自动在 `.vscode/launch.json` 中添加两个调试配置：

1. **MarkRunner CLI**：全局运行配置
2. **MarkRunner Context Debug**：上下文调试配置

#### 右键调试

1. 打开测试文件（`test_*.py`）
2. **右键**编辑器，选择 **Debug Markrunner**
3. 插件自动启动调试会话

#### 自定义调试配置

如果需要自定义调试参数：

1. 在 `.vscode/launch.json` 中创建自定义配置
2. 设置配置中的 `path.markrunner.contextLaunchConfigName` 为自定义配置名称
3. 右键调试时使用自定义配置

### 6. 刷新视图

当代码文件发生变化时：

1. 点击树形视图顶部的 **刷新** 按钮
2. 插件重新扫描和解析项目文件
3. 自动保持展开状态和搜索关键词

## 使用场景

### 场景 1：快速熟悉项目

**需求**：刚加入项目，需要了解测试元素和方法的组织结构

**步骤**：
1. 打开 PATH 侧边栏
2. 浏览测试元素树，了解有哪些页面和元素
3. 浏览测试方法树，查看已封装的测试方法
4. 双击节点查看源码，快速学习

### 场景 2：根据禅道用例编写测试

**需求**：测试人员在禅道系统中创建了新的测试用例，需要编写自动化脚本

**步骤**：
1. 在目标文件夹右键，选择 "Create Case"
2. 输入禅道用例 ID（如 `23456`）
3. 确认应用名称
4. 插件自动生成文件，包含用例标题和步骤
5. 根据步骤注释，使用 AI 或手动编写代码

### 场景 3：手动编写测试代码

**需求**：根据测试步骤手动编写自动化代码

**步骤**：
1. 创建或打开测试文件
2. 在元素树中找到需要操作的元素
3. 拖拽到编辑器，或右键选择操作类型
4. 在方法树中找到需要调用的方法
5. 拖拽到编辑器，生成调用代码

### 场景 4：使用 AI 快速生成代码

**需求**：有大量步骤需要编写，手动效率低

**步骤**：
1. 确保测试文件中有步骤注释
2. 配置 AI 服务
3. 右键选择 "AI Generation"
4. 查看辅助视图中的报告，检查生成结果
5. 手动调整生成的代码（如需要）

### 场景 5：调试失败的测试用例

**需求**：某个测试用例执行失败，需要调试

**步骤**：
1. 打开失败的测试文件
2. 在需要调试的地方设置断点
3. 右键选择 "Debug Markrunner"
4. 单步执行，查看变量值
5. 定位问题并修复

### 场景 6：查找某个方法的定义

**需求**：在代码中看到一个方法调用，想了解具体实现

**步骤**：

1. 打开方法树
2. 在搜索框中输入方法名
3. 双击搜索结果，自动跳转到定义位置
4. 查看方法实现和文档

## 最佳实践

### 1. 项目结构规范

建议的项目结构：

```
project/
├── method/              # 测试方法目录
│   ├── __init__.py
│   ├── logic/           # 模块
│   │   ├── __init__.py
│   │   ├── logic_editor/
│   │   │   ├── __init__.py
│   │   │   ├── logic_editor_ele.py
│   │   │   └── ...
│   │   └── ...
│   └── ...
├── case/                # 测试用例目录
│   ├── base_case.py
│   ├── login/
│   │   ├── test_login_001.py
│   │   └── ...
│   └── ...
└── .vscode/
    └── launch.json
```

### 2. Ele 变量命名

- **变量名**：使用英文，遵循 snake_case 命名规范
- **描述（desc）**：使用中文，简洁明了

示例：

```python
class LoginPage(Ele):
    username_input = Ele(attr={...}, desc='用户名输入框', ...)
    submit_btn = Ele(attr={...}, desc='提交按钮', ...)
```

### 3. 方法文档规范

为每个方法添加清晰的文档字符串：

```python
def login(username: str, password: str):
    """用户登录
    
    Args:
        username: 用户名
        password: 密码
    """
    # 方法实现
```

这样在方法树中可以看到完整的方法说明。

### 4. 包名中文化

在每个包的 `__init__.py` 中添加包含中文的类：

```python
class 逻辑编辑器:
    """逻辑编辑器模块"""
    pass
```

### 6. AI 使用技巧

- **步骤描述清晰**：使用精确的术语，避免模糊表达
- **关键词加权**：在步骤中使用 【】包裹重要关键词
- **检查生成结果**：AI 生成的代码可能不完全准确，需要人工检查
- **迭代优化**：根据生成结果调整步骤描述，提高准确性

## 进阶技巧

### 自定义调试配置

在 `launch.json` 中创建自定义配置，添加额外参数：

```json
{
    "name": "My Custom Debug",
    "type": "debugpy",
    "request": "launch",
    "module": "markrunner.cli",
    "args": ["run", "-p", "${workspaceFolder}", "--verbose"],
    "console": "integratedTerminal"
}
```

然后在设置中指定：

```json
{
    "path.markrunner.contextLaunchConfigName": "My Custom Debug"
}
```
