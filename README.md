# PATH 插件完整介绍

> **P**lugin **A**uto **T**est **H**omie - Python 自动化测试助手

## 📋 插件概述

**PATH** 是一款专为 Python 自动化测试开发者设计的 VSCode 扩展插件，旨在大幅提升测试代码编写效率和项目管理能力。插件智能解析 Python 代码中的 Ele 类变量和测试方法，提供直观的树形结构浏览界面，并深度集成禅道（Zentao）项目管理系统和 AI 辅助功能。

### 核心价值

- **智能解析**：自动识别和组织测试元素及方法
- **可视化浏览**：直观的树形结构，快速定位和导航代码
- **禅道集成**：一键同步测试用例，自动生成标准化测试脚本
- **AI 辅助**：智能生成测试代码，大幅降低重复性工作
- **拖拽编辑**：可视化操作，快速构建测试流程

## ✨ 核心功能特性

### 1. 🌳 双树形视图系统

#### 测试元素树 (Ele Tree)

- **自动解析**：扫描项目中所有 `*_ele.py` 文件，提取 Ele 类变量
- **层级展示**：按照 Ele 变量的层级关系（hierarchy）组织树形结构
- **中英双语**：支持中文描述（desc）和英文变量名同时显示
- **智能搜索**：实时过滤，支持中文和英文关键词搜索
- **点击跳转**：单击元素节点，自动打开对应文件并定位到变量定义行
- **状态保持**：切换文件或刷新后自动恢复展开状态和搜索关键词

#### 测试方法树 (Methods Tree)

- **包结构展示**：按照 Python 包结构组织，清晰展示 method 目录下的所有包和模块
- **中文包名映射**：读取各包 `__init__.py` 文件中的类名或属性，将英文包名替换为中文显示
- **方法列表**：展示每个模块中的所有测试方法，包含方法文档字符串
- **双击跳转**：
  - 双击文件夹节点：自动打开该包的 `__init__.py` 文件
  - 双击方法节点：跳转到方法定义位置
- **智能搜索**：支持按方法名、文档字符串、文件路径搜索

### 2. 🎯 拖拽操作与代码生成

#### 元素拖拽

- **拖拽到编辑器**：将测试元素树中的 Ele 变量拖拽到代码编辑器
- **自动生成路径**：根据元素的层级关系自动生成完整的引用路径
  - 示例：拖拽 "提交按钮" → 自动生成 `page.dialog.submit_btn`
- **智能插入**：自动在光标位置插入，保持正确的缩进

#### 右键操作菜单

在测试元素树中右键点击元素，可快速添加操作代码：

- **点击操作**：生成 `.click()` 代码
- **双击操作**：生成 `.double_click()` 代码
- **批量操作**：支持选中多个元素批量生成代码

### 3. 📦 禅道 (Zentao) 深度集成

#### 创建测试用例

通过右键菜单 "Create Case" 快速创建测试文件：

1. **智能命名**：
   - 自动扫描目标文件夹中已有的测试文件
   - 按照 `test_<folder_name>_<id>.py` 格式命名
   - 自动计算下一个可用 ID（如 `test_login_003.py`）

2. **禅道集成**：
   - 输入禅道用例 ID，自动拉取用例信息
   - 自动填充用例标题、前置条件、测试步骤
   - 将测试步骤转换为代码注释

3. **灵活模板**：
   - **完整模板**（输入禅道 ID）：包含完整的测试用例信息和步骤
   - **最小模板**（跳过禅道 ID）：生成基础测试类结构

### 4. 🤖 AI 智能辅助生成

通过右键菜单 "AI Generation" 触发 AI 辅助生成：

1. **步骤解析**：自动解析测试文件中的 "步骤" 和 "预期" 注释
2. **智能匹配**：使用 AI 理解测试步骤，智能匹配最合适的方法
3. **参数填充**：AI 自动推断方法调用所需的参数
4. **批量生成**：一次处理文件中所有未实现的步骤
5. **可视化报告**：生成 HTML 格式的执行报告

### 5. 🐞 调试支持

- **自动配置**：插件启动时自动添加 `launch.json` 调试配置
- **右键调试**：在 `test_*.py` 文件中右键选择 "Debug Markrunner"
- **自定义配置**：支持在 `launch.json` 中自定义配置

### 6. 🔍 高级搜索与导航

- **即时过滤**：输入关键词，实时显示匹配结果
- **高亮显示**：匹配的关键词高亮显示
- **自动展开**：自动展开包含匹配项的节点
- **精准跳转**：跳转到精确的行号，目标行显示在编辑器中心

## 🏗️ 技术架构

### 核心模块

- **extension.ts**：插件入口，注册所有命令和视图
- **parseEle.ts**：Ele 解析器，使用正则表达式解析 Python 文件
- **eleTreeWebviewProvider.ts**：测试元素树 Webview 提供者
- **methodsTreeWebviewProvider.ts**：测试方法树 Webview 提供者
- **zentaoService.ts**：禅道 API 集成服务
- **aiService.ts**：AI 服务集成
- **createCase.ts**：测试用例创建功能
- **launchConfig.ts**：调试配置自动管理

### 解析机制

#### Ele 变量解析

- 识别 Ele 类继承关系
- 提取变量名、描述（desc）、参数
- 解析层级结构（hierarchy）
- 支持多行变量定义

#### 方法解析

- 扫描 `method` 目录下的所有 Python 文件
- 识别方法定义和文档字符串
- 支持识别断言方法

#### 包名映射

支持以下方式：
1. 类名本身包含中文字符
2. 类属性（name、display_name、title、chinese_name、desc、description）
3. 类文档字符串

优先级：类名 > 属性值 > 文档字符串

## 🔧 配置说明

### 禅道配置

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `path.zentao.host` | 禅道服务器地址 | `10.0.18.30` |
| `path.zentao.username` | 禅道用户名 | `automation` |
| `path.zentao.password` | 禅道密码 | `******` |

### 应用配置

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `path.appName` | 默认应用名称 | `Logic` |

### 调试配置

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `path.markrunner.contextLaunchConfigName` | 右键调试使用的配置名称 | `MarkRunner Context Debug` |

### AI 配置

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `path.ai.url` | AI 模型服务 URL | `http://127.0.0.1:8045/v1` |
| `path.ai.apiKey` | AI API 密钥 | `your-api-key` |
| `path.ai.model` | AI 模型名称 | `gemini-3-flash` |

## 📦 安装与部署

### 从 VSIX 安装

1. 下载最新的 `.vsix` 文件
2. 在 VSCode 中打开命令面板（`Ctrl+Shift+P`）
3. 输入 "Install from VSIX"
4. 选择下载的 `.vsix` 文件

### 开发环境运行

```bash
git clone <repository-url>
npm install
npm run compile
# 按 F5 启动调试
```
