# PATH (Plugin Auto Test Homie) - VSCode 插件

> **P**lugin **A**uto **T**est **H**omie

## 🚀 插件概述

**PATH** 是一个专为 VSCode 设计的Python自动化测试助手插件，智能解析和显示 Python 代码中的 Ele 类变量和测试方法，集成禅道（Zentao）系统，提供强大的自动化测试代码生成和用例管理功能。插件支持中文显示、智能搜索、拖拽操作、禅道对接等现代化功能，极大提升自动化测试开发效率。

## ✨ 核心特性

### 🔌 禅道 (Zentao) 深度集成 [NEW]
- **自动获取用例**：创建测试文件时，输入禅道ID自动拉取用例标题、步骤和前置条件
- **智能模板生成**：自动将禅道步骤转换为代码注释，生成规范的测试脚本模板
- **最小化模板**：支持创建不关联禅道的纯净测试模板
- **配置灵活**：支持配置禅道连接信息及默认应用名称

### 🌳 智能树形结构
- **测试元素树**：自动解析Python代码中的Ele类变量
- **测试方法树**：展示项目中的所有测试方法
- **快速导航**：双击方法树中的文件夹可直接打开对应的 `__init__.py` 文件 [NEW]
- **双视图设计**：元素与方法并行显示，提升开发效率
- **状态保持**：切换插件后自动保持展开状态和搜索关键词

### ⚡ 自动化测试生成
- **智能创建用例**：
    - 右键文件夹或文件选择 "Create Case"
    - 自动计算文件名ID（如 `test_login_003.py`），无需手动编号 [NEW]
    - 智能识别应用名称（可通过配置跳过输入）
- **操作代码生成**：
    - 支持点击、双击等操作自动生成
    - 拖拽元素生成精确路径
    - 自动跳转到新生成的代码位置

### 🔍 高级搜索与导航
- **双语搜索**：支持中文描述和英文变量名搜索
- **实时过滤**：输入即搜索，实时显示匹配结果
- **精确跳转**：双击元素或方法自动跳转到源码位置

## ⚙️ 配置说明 [NEW]

在 VSCode 设置中（Settings > Extensions > PATH Plugin）可配置以下选项：

| 配置项 | 说明 | 示例 |
|--------|------|------|
| `path.zentao.host` | 禅道服务器地址 | `10.0.18.30` |
| `path.zentao.username` | 禅道用户名 | `automation` |
| `path.zentao.password` | 禅道密码 | `******` |
| `path.appName` | 默认应用名称 (配置后创建用例时不再询问) | `Logic` |

## 🎮 使用指南

### 1. 连接禅道
1. 打开 VSCode 设置（`Ctrl+,`）
2. 搜索 `path` 找到扩展设置
3. 填入禅道 Host, Username, Password

### 2. 创建测试用例
1. 在 VSCode 资源管理器中，右键点击目标文件夹或文件
2. 选择菜单 **Create Case** (创建用例)
3. **输入禅道ID**（可选）：
    - 输入ID（如 `12345`）：自动从禅道拉取标题和步骤生成详细模板
    - 直接回车：生成基础空模板
4. **输入应用名称**：
    - 如果已配置 `path.appName`，自动跳过此步骤
    - 否则需手动输入（如 `Login`）
5. 插件将自动生成 `test_folder_xxx.py` 文件并打开

### 3. 方法树导航
- **浏览**：展开树节点查看包结构
- **编辑包**：双击文件夹节点，自动打开该包的 `__init__.py`
- **定位方法**：双击叶子节点，跳转到测试方法定义处

### 4. 元素拖拽
1. 打开 "PATH - 测试元素树"
2. 找到目标元素（如 "提交按钮"）
3. 将其拖拽到编辑器中的 Python 代码里
4. 自动生成变量路径：`page.dialog.submit_btn`

## 📦 安装指南

### 从VSIX文件安装
1. 下载最新 `.vsix` 文件
2. 在 VSCode 命令面板输入 "Install from VSIX"
3. 选择文件进行安装

### 开发环境运行
```bash
git clone <repository-url>
npm install
npm run compile
# 按 F5 启动调试
```
